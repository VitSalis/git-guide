Οδηγός git για αρχάριους

Vitalis Salis <vitsalis@gmail.com>

Αυτό το άρθρο απευθύνεται κυρίως σε προγραμματιστές που θέλουν να χρησιμοποιήσουν το git για να μοιραστούν τον κώδικά τους, αλλά αν δεν είναι αυτός ο στόχος σας μπορείτε να χρησιμοποιήσετε το git για να μοιραστείτε και άλλου είδους πράγματα, όπως φωτογραφίες, έγγραφα, κλπ.

Γιατί να χρησιμοποιήσω το git;

Το git κάνει την ζωή μας ευκολότερη. Μας δίνει πολλές δυνατότητες όπως:

  • Ιστορικό όλων των αλλαγών στο project μας.
  • Να μπορούμε να μοιραζόμαστε δημοσίως ή ιδιωτικά τα project μας.
  • Να δουλεύουμε σε ομάδες γνωρίζοντας πότε ακριβώς και τι έκανε ο καθένας.

Σε αυτό τον οδηγό θα μάθουμε να χρησιμοποιούμε τις εντολές init, config, status, add commit, remote και push.

Τι είναι το git;

Το git είναι ένα σύστημα version control.

Version control ονομάζεται ο έλεγχος των αλλαγών σε αρχεία, φακέλους, προγράμματα, εικόνες και γενικά σε οτιδήποτε μπορούμε να αποθηκεύσουμε ή να αλλάξουμε.

Ως παράδειγμα φανταστείτε ένα αρχείο το οποίο περιέχει το μήνυμα «Μου αρέσει το παγωτό.» Έπειτα από κάποιον καιρό αποφασίζετε να αλλάξετε το μήνυμα σε «Μου αρέσει το παγωτό σοκολάτα.» Αν χρησιμοποιούσατε ένα σύστημα version control, το σύστημα αυτό θα είχε αποθηκεύσει και το πρώτο μήνυμα αλλά και το δεύτερο ώστε να μπορείτε να δείτε ποιο ήταν το μήνυμα πριν το αλλάξετε, να δείτε τις αλλαγές που έγιναν και άλλα πολλά.

Στο git χρησιμοποιείται συχνά ο όρος «εικόνα». Mια εικόνα είναι μια κατάσταση των αρχείων (ή οποιουδήποτε πράγματος αποθηκεύετε). Θα μπορούσαμε επίσης να πούμε πως μια «εικόνα» είναι τα περιεχόμενα ενός φακέλου σε μια στιγμή στον χρόνο. Δηλαδή οι υποφάκελοι, τα αρχεία και τα περιεχόμενα τους.

Στο προηγούμενο παράδειγμα αν χρησιμοποιούσαμε ένα σύστημα version control η πρώτη αποθηκευμένη εικόνα θα ήταν ένα αρχείο το οποίο θα περιείχε το μήνυμα «Μου αρέσει το παγωτό.», ενώ η δεύτερη αποθηκευμένη εικόνα θα ήταν ένα αρχείο το οποίο θα περιείχε το μήνυμα «Μου αρέσει το παγωτό σοκολάτα.» Μια εικόνα μπορεί να περιέχει πολλά αρχεία, φωτογραφίες και άλλα, όχι απαραιτήτως ξεχωριστά.

Η πιο βασική χρήση του git είναι η δημιουργία εικόνων και η αποθήκευση τους σε repositories.

Ένα git repository είναι μια αποθήκη στην οποία αποθηκεύουμε τις εικόνες μας για μελλοντική χρήση ή για να τις μοιραστούμε με τον κόσμο. Επιπλέον, μας δίνει την δυνατότητα να χρησιμοποιήσουμε όλα τα εργαλεία του git μέσα σε αυτό.

Το git το χρησιμοποιούμε από την κονσόλα (terminal) με την μορφή git «εντολή» όπου «εντολή» η εντολή που θέλουμε να εκτελέσουμε.

Μπορείτε να εγκαταστήσετε το git από το επίσημο website της ομάδας του.

Σημείωση: Οι χρήστες των windows θα κατεβάσουν το git bash. Να το χρησιμοποιούν αυτό αντί για το command line των windows στα παραδείγματα.

init

Όπως είπαμε το git αφορά την διαχείριση αρχείων και φακέλων αποθηκεύοντας την κατάσταση τους σε εικόνες, οπότε ας πάμε σε έναν φάκελο στον υπολογιστή μας που θα τον ονομάσουμε «cats».

Στον φάκελο «cats», όπως προδίδει το όνομα, θα αποθηκεύουμε πράγματα σχετικά με γάτες. Θα φτιάξουμε δύο υποφάκελους, τους «mitsos» και «giannis» οι οποίοι περιέχουν πληροφορίες για τις δυο γάτες μου, τον Μήτσο και τον Γιάννη αντίστοιχα. Κάθε υποφάκελος θα περιέχει ένα αρχείο που ονομάζεται «desc.txt» το οποίο είναι μια περιγραφή της συγκεκριμένης γάτας.

Το αρχείο «desc.txt» στον υποφάκελο «mitsos» θα περιέχει την περιγραφή «Ο Μήτσος είναι ένας υπέροχος μαύρος γάτος.», ενώ το αρχείο «desc.txt» στον υποφάκελο «giannis» θα περιέχει την περιγραφή «Ο Γιάννης αγαπάει την γυμναστική.»

Θα σας πρότεινα να φτιάξετε έναν ίδιο φάκελο με αυτόν που περιέγραψα. Αφού το κάνετε αυτό ανοίξτε το τερματικό (terminal) ή το git bash σε αυτόν τον φάκελο έτσι ώστε να μπορείτε να δοκιμάζετε τα παραδείγματα.

Αυτά είναι τα περιεχόμενα του φακέλου μας.

~/cats $ ls -R
giannis mitsos ./giannis: desc.txt ./mitsos: desc.txt
~/cats $

Ό,τι βρίσκεται πριν από το $ μας δείχνει σε ποιον φάκελο βρισκόμαστε (στην περίπτωση μας βρισκόμαστε στον φάκελο cats που έχουμε δημιουργήσει) ενώ ό,τι βρίσκεται μετά το $ είναι η εντολή που εκτελούμε. Τώρα εκτελέσαμε την εντολή ls -R.

Για όσους δεν την γνωρίζουν, η εντολή ls μας εμφανίζει τα περιεχόμενα του φακέλου στον οποίο βρισκόμαστε και αν δώσουμε την παράμετρο -R μας το κάνει αναδρομικά.

Όπως είπαμε πριν, αυτή η κατάσταση του φακέλου και των αρχείων που περιέχει είναι η εικόνα του φακέλου. Ο μόνος τρόπος για να αλλάξει η εικόνα του φακέλου θα ήταν να κάνουμε κάποια αλλαγή σε αυτόν π.χ να προσθέσουμε ένα αρχείο, να αλλάξουμε το περιεχόμενο κάποιου ήδη υπάρχοντος κλπ.

Έναν φάκελο στον υπολογιστή μας μπορούμε να τον κάνουμε ένα git repository έτσι ώστε να μπορούμε να χρησιμοποιήσουμε όλα τα εργαλεία του git μέσα σε αυτόν.

Για να κάνουμε έναν φάκελο git repository χρησιμοποιούμε την εντολή git init.

~/cats $ git init
Initialized empty Git repository in /home/your_username/cats/.git/
~/cats $

Η έξοδος (μπλε γράμματα) μας λέει πως ο φάκελος μας έγινε επιτυχώς ένα άδειο (χωρίς καμία αποθηκευμένη εικόνα) git repository δίνοντάς μας πρόσβαση σε όλα τα εργαλεία του git.

config

Κάθε εικόνα που αποθηκεύεται περιέχει επίσης το όνομα και το email αυτού που την δημιούργησε. Ο λόγος που γίνεται αυτό είναι ώστε εσείς και οι συνεργάτες σας να μπορείτε να βλέπετε ποιος έκανε τι.

Οπότε πρέπει να ρυθμίσουμε στο git το όνομα και το email μας ώστε να συμπεριληφθούν στις εικόνες που θα αποθηκεύσουμε.

Αυτό το κάνουμε με την εντολή git config.

Η εντολή αυτή όμως ορίζει τις ρυθμίσεις μόνο στο git repository στο οποίο βρισκόμαστε τώρα. Για να έχουμε τις ρυθμίσεις αυτές σε κάθε repository που δημιουργούμε και χρησιμοποιούμε δίνουμε την παράμετρο --global.

Για να ορίσουμε το ονοματεπώνυμο μας χρησιμοποιούμε την παράμετρο user.name "Your FullName".

~/cats $ git config --global user.name "Your FullName"

Όπου Your FullName το ονοματεπώνυμο σας (στην περίπτωση μου Vitalis Salis).

Για να ορίσουμε το email μας χρησιμοποιούμε την παράμετρο user.email "Your_Email".

~/cats $ git config --global user.email "Your_Email"

Όπου Your_Email το email σας (στην περίπτωση μου vitsalis@gmail.com).

add

Στο git για να μπορείς να αποθηκεύσεις μια εικόνα των αρχείων σου πρέπει να τα προσθέσεις στην staging area.

Η staging area είναι μια περιοχή η οποία περιέχει πληροφορίες για το από τι θα αποτελείται η επόμενη εικόνα που θα αποθηκευτεί.

Είναι πολύ σημαντική, γιατί ο μόνος τρόπος που μπορούμε να αποθηκεύσουμε τα αρχεία ή τους φακέλους μας σε μια εικόνα, είναι να έχουν ήδη προστεθεί αυτά στην staging area.

Φυσικά, όταν προσθέτουμε έναν φάκελο στην staging area προσθέτουμε και όλα τα αρχεία και υποφάκελους τα οποία περιέχει.

Αυτή τη στιγμή η staging area είναι άδεια διότι δεν έχουμε προσθέσει τίποτα εκεί.

Εκτελώντας την εντολή git status -s μπορούμε να δούμε ποια αρχεία έχουν προστεθεί στην staging area και ποια όχι.

~/cats $ git status -s
?? giannis/ ?? mitsos/
~/cats $

Τα ερωτηματικά δείχνουν πως οι φάκελοι «mitsos» και «giannis» δεν έχουν προστεθεί ποτέ στο git και συνεπώς στην staging area.

Αφού επιβεβαιώσαμε ότι οι φάκελοί μας δεν έχουν προστεθεί στην staging area, ας τους προσθέσουμε.

Για να τους προσθέσουμε χρησιμοποιούμε την εντολή git add.

Στην εντολή add μπορούμε να δώσουμε ως παράμετρο το όνομα του αρχείου ή φακέλου που θέλουμε να προσθέσουμε ή . για να προσθέσουμε όλα τα αρχεία και φακέλους που υπάρχουν στον φάκελο που βρισκόμαστε.

Εγω τώρα θα χρησιμοποιήσω την παράμετρο .

~/cats $ git add . ~/cats $

Ας επιβεβαιώσουμε ότι οι φάκελοί μας προστέθηκαν στην staging area τρέχοντας ξανά την εντολή git status -s.

~/cats $ git status -s
A giannis/desc.txt A mitsos/desc.txt
~/cats $

Τώρα δίπλα από κάθε αρχείο εμφανίζεται ένα «Α» από το «Added» που σημαίνει ότι τα αρχεία έχουν προστεθεί στην staging area.

Θα μπορούσαμε φυσικά να προσθέσουμε κάθε φάκελο ξεχωριστά αντί να χρησιμοποιήσουμε την παράμετρο . παίρνοντας το ίδιο αποτέλεσμα.

~/cats $ git add mitsos ~/cats $ git add giannis ~/cats $ git status -s
A giannis/desc.txt A mitsos/desc.txt
~/cats $

commit

Δεν έχουμε αποθηκεύσει όμως καμία εικόνα ακόμα. Έχουμε τις πληροφορίες για την εικόνα που θέλουμε να αποθηκεύσουμε στην staging area, οπότε το μόνο που μας μένει είναι να την αποθηκεύσουμε.

Για να το κάνουμε αυτό θα χρησιμοποιήσουμε την εντολή git commit.

Η εντολή git commit χρησιμοποιεί τις πληροφορίες για τα αρχεία που υπάρχουν στην staging area για να δημιουργήσει μια νέα εικόνα την οποία αποθηκεύει μόνιμα στο git repository. Μπορεί να πάρει ως παράμετρο -m "message" όπου message το μήνυμα που εξηγεί τις αλλαγές από την προηγούμενη αποθηκευμένη εικόνα. Αυτό μας δίνει την δυνατότητα να μπορούμε να εξηγούμε τις αλλαγές που κάναμε στην ομάδα μας για να κάνουμε την συνεργασία ευκολότερη και λιγότερο χρονοβόρα.

Αν δεν δοθεί η παράμετρος -m τότε θα ανοίξει καινούριο παράθυρο όπου εκεί θα γράψετε το message.

Προς το παρόν το git δεν έχει καμία αποθηκευμένη εικόνα, οπότε ας δημιουργήσουμε την πρώτη.

~/cats $ git commit -m "Add mitsos and giannis"
[master (root-commit) 445eeae] Add mitsos and giannis 2 files changed, 2 insertions(+) create mode 100644 giannis/desc.txt create mode 100644 mitsos/desc.txt
~/cats $

Εκτελώντας τώρα git status -s βλέπουμε πως δεν υπάρχει έξοδος.

~/cats $ git status -s ~/cats $

Αυτό συμβαίνει γιατί η εντολή git status μας εμφανίζει τι μπορεί να προστεθεί στην staging area ή αν αυτή περιέχει πληροφορίες για να δημιουργηθεί μια νέα εικόνα.

Για αυτόν τον λόγο, αφού δημιουργήσαμε μια νέα εικόνα με τις πληροφορίες που υπήρχαν στην staging area και δεν υπάρχουν καινούριες αλλαγές να προστεθούν σε αυτή, η εντολή git status -s δεν μας δίνει καμία έξοδο.

H παράμετρος -s που χρησιμοποιήσαμε στην διάρκεια αυτού του οδηγού απλά λέει στην εντολή git status να μας εμφανίσει μια σύντομη περιγραφή της εξόδου που θα παίρναμε αν εκτελούσαμε απλά git status.

Ας κάνουμε κάποιες αλλαγές στα αρχεία μας για εξάσκηση.

Θα χρησιμοποιήσω την εντολή cat για να δω τα περιεχόμενα της περιγραφής του Μήτσου.

~/cats $ cat mitsos/desc.txt
O Μήτσος είναι ένας υπέροχος μαύρος γάτος.
~/cats $

Αλλά ο Μήτσος είναι και έξυπνος. Ας το προσθέσω και αυτό στην περιγραφή. Θα χρησιμοποιήσω τον text editor vi για να κάνω αυτή την αλλαγή. Φυσικά εσείς μπορείτε να χρησιμοποιήσετε όποιον text editor θέλετε. Επίσης μπορείτε αντί για το τερματικό ή το git bash να χρησιμοποιήσετε την διεπαφή του λειτουργικού σας συστήματος. Αφού κάνω τις αλλαγές μου η περιγραφή θα αλλάξει σε «O Μήτσος είναι ένας υπέροχος και έξυπνος μαύρος γάτος.»

~/cats $ vi mitsos/desc.txt ~/cats $ cat mitsos/desc.txt
O Μήτσος είναι ένας υπέροχος και έξυπνος μαύρος γάτος.
~/cats $

Τώρα που έγιναν οι αλλαγές αν εκτελέσουμε την εντολή git status -s θα πάρουμε αυτή την έξοδο.

~/cats $ git status -s
M mitsos/desc.txt
~/cats $

Το M από το «Modified» σημαίνει πως έχουν γίνει αλλαγές σε αυτό το αρχείο, οπότε υπάρχουν αλλαγές που μπορούν να προστεθούν στην staging area.

Ας χρησιμοποιήσουμε την εντολή git add για να προσθέσουμε την αλλαγή στο περιεχόμενο του αρχείου «desc.txt» που βρίσκεται στον φάκελο «mitsos» στην staging area και ας δημιουργήσουμε με τις καινούριες πληροφορίες που απέκτησε η staging area μια εικόνα η οποία θα αποθηκευτεί στο git repository μας με την εντολή git commit.

~/cats $ git add mitsos/desc.txt ~/cats $ git status -s
M mitsos/desc.txt
~/cats $ git commit -m "Add a new description for mitsos"
[master 3d39595] Add a new description for mitsos 1 file changed, 1 insertion(+), 1 deletion(-)
~/cats $ git status -s ~/cats $

Αν κοιτάξετε την έξοδο καλύτερα θα παρατηρήσετε πως πριν προσθέσουμε τις αλλαγές στην staging area το «Μ» που εμφανιζόταν δίπλα από το όνομα του αρχείου όταν εκτελούσαμε την εντολή git status -s ήταν κόκκινο, ενώ αφού το προσθέσαμε έγινε πράσινο. Αυτός είναι ο τρόπος του git να μας δείχνει ποια αρχεία ή ποιες αλλαγές έχουν προστεθεί στην staging area και ποια/ποιες όχι. Αν εσάς δεν σας εμφανίζονται χρώματα μην πανικοβληθείτε, δεν εμφανίζονται σε όλους.

Η αλήθεια είναι πως δεν μου αρέσει ο Γιάννης, οπότε λέω να τον διαγράψω απ' την λίστα με τις γάτες μου.

Για να το κάνω αυτό έχω δύο επιλογές.

Να χρησιμοποιήσω την εντολή git rm ή την εντολή rm.

Η εντολή git rm έχει το πλεονέκτημα πως, αν την εκτελέσω, οι αλλαγές προστίθενται κατευθείαν στην staging area.

~/cats $ git rm -r giannis
rm 'giannis/desc.txt'
~/cats $ git status -s
D 'giannis/desc.txt'
~/cats $ git commit -m "Remove giannis"
[master 8c67bb3] Remove giannis 1 files changed, 1 deletion(-) delete mode 100644 giannis/desc.txt
~/cats $ git status -s ~/cats $

Το «D» από το «Deleted» σημαίνει πως το αρχείο έχει διαγραφεί.

Όπως βλέπετε τα αρχείο διαγράφηκε και οι αλλαγές προστέθηκαν στην staging area. Έπειτα αποθηκεύσαμε τις πληροφορίες απο την staging area σε μια εικόνα με την εντολή git commit.

Έχουμε όμως και την εναλλακτική να χρησιμοποιήσουμε την εντολή rm.

Αν χρησιμοποιήσατε την εντολή git rm ήδη, δεν υπάρχει νοήμα να χρησιμοποιήσετε την παρακάτω εντολή διότι τα αρχεία έχουν ήδη διαγραφεί.

Αν χρησιμοποιήσουμε την εντολή rm θα πρέπει να προσθέσουμε τις αλλαγές μόνοι μας στην staging area.

~/cats $ rm -r giannis ~/cats $ git status -s
D giannis/desc.txt
~/cats $ git add giannis
warning: You ran 'git add' with neither '-A (--all)' or '--ignore-removal', whose behaviour will change in Git 2.0 with respect to paths you removed. Paths like 'giannis/desc.txt' that are removed from your working tree are ignored with this version of Git. * 'git add --ignore-removal ', which is the current default, ignores paths you removed from your working tree. * 'git add --all ' will let you also record the removals. Run 'git status' to check the paths you removed from your working tree.
~/cats $ git status -s
D giannis/desc.txt
~/cats $

Εκτελούμε μετά την διαγραφή git status -s και επιβεβαιώνουμε οτι η διαγραφή έγινε και πως οι αλλαγές δεν έχουν προστεθεί στην staging area (κόκκινο «D»).

Έπειτα εκτελούμε git add giannis για να προσθέσουμε τις αλλαγές αυτές στην staging area αλλά μας εμφανίζεται μια προειδοποιήση.

Αν εκτελέσουμε ξανά git status -s βλέπουμε πως δεν έχει αλλάξει τίποτα με την εντολή git add giannis.

Αυτό συμβαίνει γιατί το git για λόγους ασφαλείας δεν μας αφήνει να προσθέσουμε τις διαγραφές αν δεν έχουμε χρησιμοποιήσει την παράμετρο -A απο το «All». Σε αυτό οφείλεται η προειδοποιήση που είδαμε πριν.

Ας δοκιμάσουμε να προσθέσουμε τις διαγραφές ξανά αλλά αυτήν την φορά με την εντολή git add giannis -A.

~/cats $ git add giannis -A ~/cats $ git status -s
D giannis/desc.txt
~/cats $ git commit -m "Remove giannis"
[master 8c67bb3] Remove giannis 1 files changed, 1 deletion(-) delete mode 100644 giannis/desc.txt
~/cats $ git status -s ~/cats $

remote

Τώρα που αποθηκεύσαμε διάφορες εικόνες στο git repository μας συνήθως θα θέλαμε να τις ανεβάσουμε σε ένα online git repository.

Αυτό έχει πολλά πλεονεκτήματα. Μερικά από αυτά είναι:

  • Μπορούμε να μοιραστούμε το repository μας με οποιονδήποτε επιθυμούμε.
  • Απομακρυσμένη εργασία, διότι οποιοσδήποτε συνάδελφος μας έχει συνδέση στο internet μπορεί να ανεβάζει την δουλεία του στο online git repository απο οπουδήποτε.
  • Κατανόηση των αλλαγών που οι συνάδελφοι μας έκαναν στον κώδικα μέσω των messages που εξηγούν τις αλλαγές στις αποθηκευμένες εικόνες, χωρίς να χρειάζεται να κάνουμε οι ίδιοι αυτή την εξήγηση.
  • Αν επιλέξουμε να είναι το repository μας δημόσιο δίνουμε την ευκαιρία σε οποιονδήποτε να συμμετέχει και να βοηθήσει στο project μας.

Σε αυτόν τον οδηγό θα χρησιμοποιήσω το GitHub.

Για να χρησιμοποιήσετε το GitHub θα πρέπει να ακολουθήσετε τα παρακάτω βήματα:

  1. Δημιουργήστε λογαριασμό.
  2. Δημιουργήστε ένα online git repository.
  3. Δημιουργήστε ένα ssh key και προσθέστε το στον λογαριασμό σας στο GitHub, ώστε να μπορείτε να ανεβάζετε μόνο εσείς τις εικόνες σας στο οnline git repository σας.

Κάθε online git repository έχει ένα URL. Το URL ενός online git repository μοιάζει με διεύθυνση ιστοσελίδας, αλλά αναφέρεται σε ένα online git repository και μέσω αυτού το git μπορεί να συχρονίζει τοπικό με online repository.

Όλα τα git repositories είναι ίσα. Εμείς απλά τα συγχρονίζουμε μεταξύ τους. Αυτό είναι χρήσιμο ώστε να μπορούμε να έχουμε περισσότερα από ένα online git repositories. Για να μην τυπώνουμε το URL του online git repository που θέλουμε να χρησιμοποιήσουμε κάθε φορά, το git μας επιτρέπει να αποθηκεύσουμε ένα σύντομο όνομα για κάθε URL ενός online git repository που θέλουμε να χρησιμοποιήσουμε. Αυτός ο συνδυασμός όνομα-URL είναι ένα remote.

Αν εκτελέσουμε την εντολή git remote μας εμφανίζει μια λίστα από τα ονόματα των remotes που έχει το git repository μας.

~/cats $ git remote ~/cats $

Όπως βλέπετε δεν μας βγάζει κανένα όνομα γιατί δεν έχουμε δημιουργήσει κανένα remote. Για να δημιουργήσουμε ένα χρησιμοποιούμε την εντολή git remote add [remote_name] [url] όπου remote_name το όνομα που δίνουμε στο remote και url το URL του online git repository.

Για να βρείτε το URL του online git repository σας στο GitHub, απλά πηγαίντε στα online repositories σας μέσω της διεπαφής της ιστοσελίδας του και θα σας το εμφανίσει εκεί.

Σε αυτό τον οδηγό θα χρησιμοποιήσω ως url το URL του repository git-guide που έχω δημιουργήσει στον GitHub λογαριασμό μου (VitSalis) και ως remote_name το όνομα «origin».

Προσοχή! Μην χρησιμοποιήσετε το URL του δικού μου online git repository, αλλά αυτού που δημιουργήσατε.

~/cats $ git remote add origin https://github.com/VitSalis/git-guide.git ~/cats $ git remote
origin
~/cats $ git remote -v
origin https://github.com/VitSalis/git-guide.git origin https://github.com/VitSalis/git-guide.git
~/cats $

Αν δώσουμε την παράμετρο -v στην εντολή git remote τότε μας δείχνει εκτός από το όνομα του remote και το URL του.

Τώρα έχουμε ένα online git repository και έχουμε φτιάξει ένα remote. Το μόνο που μας μένει είναι να ανεβάσουμε τις αποθηκευμένες εικόνες μας μέσω αυτού.

push

Αν επισκεφθείτε το online git repository σας θα δείτε ότι είναι άδειο. Αυτό συμβαίνει γιατί δεν έχετε ανεβάσει ακόμα το repository σας εκεί.

Σε αυτό χρησιμεύει η εντολή git push [remote_name] master όπου remote_name το όνομα του remote που περιέχει το URL του online git repository όπου θέλουμε να ανέβουν οι εικόνες.

Ως remote_name θα χρησιμοποιήσω «origin», που είναι το όνομα του remote που φτιάξαμε πριν.

Η εντολή push απλά ανεβάζει το τοπικό repository στο online git repository.

Κάθε φορά που δημιουργείτε καινούριες εικόνες, για να ανέβουν στο online git repository σας θα πρέπει να εκτελείτε την εντολή git push.

~/cats $ git push origin master
Counting objects: 8, done. Delta compression using up to 4 threads. Compressing objects: 100% (8/8), done. Writing objects: 100% (8/8), 35.58 KiB | 0 bytes/s, done. Total 8 (delta 0), reused 0 (delta 0) To git@github.com:VitSalis/git-guide.git * [new branch] master -> master
~/cats $

Αν επισκεφθείτε το online git repository σας τώρα θα δείτε την τελευταία εικόνα που αποθηκεύσατε και αν κοιτάξετε τα commits θα δείτε όλες τις εικόνες που είχατε αποθηκεύσει τοπικά, με τα αντίστοιχα μηνύματα.

Αυτό ήταν! Τώρα μπορώ να μοιραστώ τις γάτες μου με όλο τον κόσμο!

Ευχαριστώ που διαβάσατε αυτόν τον οδηγό. Τώρα ξέρετε τι είναι ένα git repository και πως να χρησιμοποιείτε τα βασικά εργαλεία του git, ώστε να μπορείτε να κάνετε την συνεργασία σας με άλλους αλλά και την ζωή σας ευκολότερη.

Θα προσπαθήσω να προσθέσω με την πάροδο του χρόνου επιπλέον ενότητες σε αυτόν τον οδηγό, γιατί το git περιέχει πολλά ακόμα ενδιαφέρον εργαλεία.
