Οδηγός git για αρχάριους

Vitalis Salis <vitsalis@gmail.com>

Αυτό το άρθρο απευθύνεται κυρίως σε προγραμματιστές που θέλουν να χρησιμοποιήσουν το git για να μοιραστούν τον κωδικά τους, αλλά αν δεν είναι αυτός ο στόχος σας μπορείτε να χρησιμοποιήσετε το git για να μοιραστείτε και άλλου είδους πράγματα, όπως φωτογραφίες, έγγραφα, κλπ.

Γιατί να χρησιμοποιήσω το git;

Το git κάνει την ζωή μας ευκολότερη. Μας δίνει πολλές δυνατότητες όπως:

  • Ευκολία στην χρήση.
  • Ιστορικό όλων των αλλαγών στο project μας.
  • Να μπορούμε να μοιραζόμαστε δημοσίως ή ιδιωτικά τα project μας.
  • Να δουλεύουμε σε ομάδες γνωρίζοντας πότε ακριβώς και τι έκανε ο καθένας.

Σε αυτό τον οδηγό θα μάθουμε να χρησιμοποιούμε τις εντολές init, status, add commit, remote και push.

Τι είναι το git;

Το git είναι ένα σύστημα version control.

Version control ονομάζεται ο έλεγχος των αλλαγών σε αρχεία, φακέλους, προγράμματα, εικόνες και γενικά σε οτιδήποτε μπορούμε να αποθηκεύσουμε ή να αλλάξουμε.

Ως παράδειγμα φανταστείτε ένα αρχείο το οποίο περιέχει το μήνυμα «Μου αρέσει το παγωτό.» Έπειτα από κάποιον καιρό αποφασίζετε να αλλάξετε το μήνυμα σε «Μου αρέσει το παγωτό σοκολάτα.» Αν χρησιμοποιούσατε ένα σύστημα version control, το σύστημα αυτό θα είχε αποθηκεύσει και το πρώτο μήνυμα αλλά και το δεύτερο ώστε να μπορείτε να δείτε ποιο ήταν το μήνυμα πριν το αλλάξετε, να δείτε τις αλλαγές που έγιναν και άλλα πολλά.

Στο git χρησιμοποιείται συχνά ο όρος «εικόνα». Mια εικόνα είναι μια κατάσταση των αρχείων (ή οποιουδήποτε πράγματος αποθηκεύετε). Θα μπορούσαμε επίσης να πούμε πως μια «εικόνα» είναι τα περιεχόμενα ενός φακέλου σε μια στιγμή στον χρόνο. Δηλαδή οι υποφάκελοι, τα αρχεία και τα περιεχόμενα τους.

Στο προηγούμενο παράδειγμα αν χρησιμοποιούσαμε ένα σύστημα version control η πρώτη αποθηκευμένη εικόνα θα ήταν ένα αρχείο το οποίο θα περιείχε το μήνυμα «Μου αρέσει το παγωτό.», ενώ η δεύτερη αποθηκευμένη εικόνα θα ήταν ένα αρχείο το οποίο θα περιείχε το μήνυμα «Μου αρέσει το παγωτό σοκολάτα.» Μια εικόνα μπορεί να περιέχει πολλά αρχεία, εικόνες και άλλα, όχι απαραιτήτως ξεχωριστά.

Η πιο βασική χρήση του git είναι η δημιουργία εικόνων και η αποθήκευση τους σε repositories.

Ένα repository είναι μια αποθήκη στην οποία αποθηκεύουμε τις εικόνες μας για μελλοντική χρήση ή για να τις μοιραστούμε με τον κόσμο.

Το git το χρησιμοποιούμε από την κονσόλα (terminal) με την μορφή git «εντολή» όπου «εντολή» η εντολή που θέλουμε να εκτελέσουμε.

Μπορείτε να εγκαταστήσετε το git από το επίσημο website της ομάδας του.

init

Όπως είπαμε το git αφορά την διαχείριση αρχείων και φακέλων αποθηκεύοντας την κατάσταση τους σε εικόνες, οπότε ας πάμε σε έναν φάκελο στον υπολογιστή μας που θα τον ονομάσουμε «cats».

Στον φάκελο «cats», όπως προδίδει το όνομα, θα αποθηκεύουμε πράγματα σχετικά με γάτες. Θα φτιάξουμε δύο υποφάκελους, τους «mitsos» και «giannis» οι οποίοι περιέχουν πληροφορίες για τις δυο γάτες μου, τον Μήτσο και τον Γιάννη αντίστοιχα. Κάθε υποφάκελος θα περιέχει ένα αρχείο που ονομάζεται «desc.txt» το οποίο είναι μια περιγραφή της συγκεκριμένης γάτας και ένα δεύτερο αρχείο το οποίο είναι μια εικόνα της γάτας.

Αυτά είναι τα περιεχόμενα του φακέλου μας.

~/cats $ ls -R
giannis mitsos ./giannis: desc.txt giannis.jpg ./mitsos: desc.txt mitsos.jpg
~/cats $

Για όσους δεν την γνωρίζουν, η εντολή ls μας εμφανίζει τα περιεχόμενα του φακέλου στον οποίο βρισκόμαστε και αν δώσουμε την παράμετρο -R μας το κάνει αναδρομικά.

Αυτή η κατάσταση του φακέλου και των αρχείων που περιέχει, όπως είπαμε πριν, είναι η εικόνα του φακέλου. Ο μόνος τρόπος για να αλλάξει η εικόνα του φακέλου θα ήταν να κάνουμε κάποια αλλαγή σε αυτόν π.χ να προσθέσουμε ένα αρχείο, να αλλάξουμε το περιεχόμενο κάποιου ήδη υπάρχοντος κλπ.

Έναν φάκελο στον υπολογιστή μας μπορούμε να τον κάνουμε ένα git repository. Όταν ο φάκελος μας είναι ένα git repository έχουμε όλα τα εργαλεία του git για αυτόν τον φάκελο.

Για να κάνουμε έναν φάκελο git repository χρησιμοποιούμε την εντολή git init.

~/cats $ git init
Initialized empty Git repository in /home/your_username/cats/.git/
~/cats $

Η έξοδος μας λέει πως ο φάκελος μας έγινε επιτυχώς ένα άδειο (χωρίς καμία αποθηκευμένη εικόνα) git repository δίνοντάς μας πρόσβαση σε όλα τα εργαλεία του git.

add

Στο git για να αποθηκεύσεις μια εικόνα των αρχείων σου πρέπει να τα περάσεις στην staging area.

Η staging area είναι ένα αρχείο, συνήθως αποθηκευμένο μέσα στο git repository το οποίο περιέχει πληροφορίες για το από τι θα αποτελείται η επόμενη εικόνα που θα αποθηκευτεί.

Είναι πολύ σημαντική, γιατί ο μόνος τρόπος που μπορούμε να αποθηκεύσουμε τα αρχεία ή τους φακέλους μας σε μια εικόνα, είναι να έχουν ήδη προστεθεί αυτά στην staging area.

Φυσικά, όταν προσθέτουμε έναν φάκελο στην staging area προσθέτουμε και όλα τα αρχεία και υποφάκελους τα οποία περιέχει.

Αυτή τη στιγμή η staging area είναι άδεια διότι δεν έχουμε προσθέσει τίποτα εκεί.

Εκτελώντας την εντολή git status -s μπορούμε να δούμε ποια αρχεία έχουν προστεθεί στην staging area και ποια όχι.

~/cats $ git status -s
?? giannis/ ?? mitsos/
~/cats $

Τα ερωτηματικά δείχνουν πως οι φάκελοι «mitsos» και «giannis» δεν έχουν προστεθεί στην staging area.

Αφού επιβεβαιώσαμε ότι οι φάκελοί μας δεν έχουν προστεθεί στην staging area, ας τους προσθέσουμε.

Για να τους προσθέσουμε χρησιμοποιούμε την εντολή git add.

Στην εντολή add μπορούμε να δώσουμε το όνομα του αρχείου ή φακέλου που θέλουμε να προσθέσουμε ή . για να προσθέσουμε όλα τα αρχεία και φακέλους που δεν έχουν προστεθεί ήδη.

~/cats $ git add . ~/cats $

Ας επιβεβαιώσουμε ότι οι φάκελοι μας προστέθηκαν στην staging area τρέχοντας ξανά την εντολή git status -s.

~/cats $ git status -s
A giannis/desc.txt A giannis/giannis.jpg A mitsos/desc.txt A mitsos/mitsos.jpg
~/cats $

Τώρα δίπλα από κάθε αρχείο εμφανίζεται ένα «Α» απο το «Added» που σημαίνει ότι τα αρχεία έχουν προστεθεί στην staging area.

Θα μπορούσαμε φυσικά να προσθέσουμε κάθε φάκελο ξεχωριστά αντί να χρησιμοποιήσουμε την παράμετρο . παίρνοντας το ίδιο αποτέλεσμα.

~/cats $ git add mitsos ~/cats $ git add giannis ~/cats $ git status -s
A giannis/desc.txt A giannis/giannis.jpg A mitsos/desc.txt A mitsos/mitsos.jpg
~/cats $

commit

Δεν έχουμε αποθηκεύσει όμως καμία εικόνα ακόμα. Έχουμε τις πληροφορίες για την εικόνα που θέλουμε να αποθηκεύσουμε στην staging area, οπότε το μόνο που μας μένει είναι να την αποθηκεύσουμε.

Για να το κάνουμε αυτό θα χρησιμοποιήσουμε την εντολή git commit.

Η εντολή commit χρησιμοποιεί τις πληροφορίες για τα αρχεία που υπάρχουν στην staging area για να δημιουργήσει μια νέα εικόνα την οποία αποθηκεύει μόνιμα στο git repository.

Η εντολή git commit μπορεί να πάρει ως παράμετρο -m 'message' όπου message το μήνυμα που εξηγεί τις αλλαγές από την προηγούμενη αποθηκευμένη εικόνα. Αυτό μας δίνει την δυνατότητα να μπορούμε να εξηγούμε τις αλλαγές που κάναμε στην ομάδα μας για να κάνουμε την συνεργασία ευκολότερη και λιγότερο χρονοβόρα.

Αν δεν δοθεί η παράμετρος -m τότε θα ανοίξει καινούριο παράθυρο όπου εκεί θα γράψετε το message.

Προς το παρόν το git δεν έχει καμία αποθηκευμένη εικόνα, οπότε ας δημιουργήσουμε την πρώτη.

~/cats $ git commit -m 'Add mitsos and giannis'
[master (root-commit) 445eeae] Add my favourite cats 4 files changed, 2 insertions(+) create mode 100644 giannis/desc.txt create mode 100644 giannis/giannis.jpg create mode 100644 mitsos/desc.txt create mode 100644 mitsos/mitsos.jpg
~/cats $

Εκτελώντας τώρα git status -s βλέπουμε πως δεν υπάρχει έξοδος.

~/cats $ git status -s ~/cats $

Αυτό συμβαίνει γιατί η εντολή git status μας εμφανίζει τι μπορεί να προστεθεί στην staging area ή αν αυτή περιέχει πληροφορίες για να δημιουργηθεί μια νέα εικόνα.

Για αυτόν τον λόγο αφού δημιουργήσαμε μια νέα εικόνα με τις πληροφορίες που υπήρχαν στην staging area και δεν υπάρχουν καινούριες αλλαγές να προστεθούν σε αυτή, η εντολή git status -s δεν μας δίνει καμία έξοδο.

H παράμετρος -s που χρησιμοποιήσαμε στην διάρκεια αυτού του οδηγού απλά λέει στην εντολή git status να μας εμφανίσει μια σύντομη περιγραφή της εξόδου που θα παίρναμε αν την εκτελούσαμε χωρίς την παράμετρο -s.

Ας κάνουμε κάποιες αλλαγές στα αρχεία μας για εξάσκηση.

Θα χρησιμοποιήσω την εντολή cat για να δω τα περιεχόμενα της περιγραφής του Μήτσου.

~/cats $ cat mitsos/desc.txt
O Μήτσος είναι ένας υπέροχος μαύρος γάτος.
~/cats $

Αλλά ο Μήτσος είναι και έξυπνος. Ας το προσθέσω και αυτό στην περιγραφή. Θα χρησιμοποιήσω τον text editor vi για να κάνω αυτή την αλλαγή.

~/cats $ vi mitsos/desc.txt ~/cats $ cat mitsos/desc.txt
O Μήτσος είναι ένας υπέροχος και έξυπνος μαύρος γάτος.
~/cats $

Αν εκτελέσουμε την εντολή git status -s θα πάρουμε αυτή την έξοδο.

~/cats $ git status -s
M mitsos/desc.txt
~/cats $

Το M από το «Modified» σημαίνει πως έχουν γίνει αλλαγές σε αυτό το αρχείο, οπότε υπάρχουν αλλαγές που μπορούν να προστεθούν στην staging area.

Ας χρησιμοποιήσουμε την εντολή git add για να προσθέσουμε την αλλαγή στο περιεχόμενο του αρχείου «desc.txt» που βρίσκεται στον φάκελο «mitsos» στην staging area και ας δημιουργήσουμε με τις καινούριες πληροφορίες που απέκτησε η staging area μια εικόνα η οποία θα αποθηκευτεί στο git repository μας με την εντολή git commit.

~/cats $ git add mitsos/desc.txt ~/cats $ git status -s
M mitsos/desc.txt
~/cats $ git commit -m 'Add a new description for mitsos'
[master 3d39595] Add a new description for mitsos 1 file changed, 1 insertion(+), 1 deletion(-)
~/cats $ git status -s ~/cats $

Αν κοιτάξετε την έξοδο καλύτερα θα παρατηρήσετε πως πριν προσθέσουμε τις αλλαγές στην staging area το «Μ» που εμφανιζόταν δίπλα από το όνομα του αρχείου όταν εκτελούσαμε την εντολή git status -s ήταν κόκκινο, ενώ αφού το προσθέσαμε έγινε πράσινο. Αυτός είναι ο τρόπος του git να μας δείχνει ποια αρχεία ή ποιες αλλαγές έχουν προστεθεί στην staging area και ποια/ποιες όχι.

Η αλήθεια είναι πως δεν μου αρέσει ο Γιάννης, οπότε λέω να τον διαγράψω απ' την λίστα με τις γάτες μου.

Για να το κάνω αυτό έχω δύο επιλογές.

Να χρησιμοποιήσω την εντολή git rm ή την εντολή rm.

Η εντολή git rm έχει το πλεονέκτημα πως αν την εκτελέσω οι αλλαγές προστίθενται κατευθείαν στην staging area.

~/cats $ git rm -R giannis
rm 'giannis/desc.txt' rm 'giannis/giannis.jpg'
~/cats $ git commit -m 'Remove giannis'
[master 8c67bb3] Remove giannis 2 files changed, 1 deletion(-) delete mode 100644 giannis/desc.txt delete mode 100644 giannis/giannis.jpg
~/cats $

Αν χρησιμοποιήσουμε την εντολή rm θα πρέπει να προσθέσουμε τις αλλαγές μόνοι μας στην staging area.

~/cats $ rm -R giannis ~/cats $ git status -s
D giannis/desc.txt D giannis/giannis.jpg
~/cats $ git add .
warning: You ran 'git add' with neither '-A (--all)' or '--ignore-removal', whose behaviour will change in Git 2.0 with respect to paths you removed. Paths like 'giannis/desc.txt' that are removed from your working tree are ignored with this version of Git. * 'git add --ignore-removal ', which is the current default, ignores paths you removed from your working tree. * 'git add --all ' will let you also record the removals. Run 'git status' to check the paths you removed from your working tree.
~/cats $ git status -s
D giannis/desc.txt D giannis/giannis.jpg
~/cats $

Το «D» από το «Deleted» σημαίνει πως το αρχείο έχει διαγραφεί.

Βλέπουμε πως η εντολή git add . δεν δούλεψε.

Αυτό συμβαίνει γιατί το git για λόγους ασφαλείας δεν μας αφήνει να προσθέσουμε τις διαγραφές αν δεν έχουμε χρησιμοποιήσει την παράμετρο -A.

Ας δοκιμάσουμε να προσθέσουμε τις διαγραφές ξανά αλλά αυτήν την φορά με την εντολή git add . -A.

~/cats $ git add . -A ~/cats $ git status -s
D giannis/desc.txt D giannis/giannis.jpg
~/cats $ git commit -m 'Remove giannis'
[master 8c67bb3] Remove giannis 2 files changed, 1 deletion(-) delete mode 100644 giannis/desc.txt delete mode 100644 giannis/giannis.jpg
~/cats $

remote

Τώρα που αποθηκεύσαμε διάφορες εικόνες στο git repository μας συνήθως θα θέλαμε να τις ανεβάσουμε σε ένα online git repository.

Αυτό έχει πολλά πλεονεκτήματα. Μερικά από αυτά είναι:

  • Μπορούμε να μοιραστούμε τις εικόνες μας με όποιον επιθυμούμε.
  • Εύκολη ανταλλαγή κώδικα με τους συναδέλφους μας.
  • Απομακρυσμένη εργασία.
  • Κατανόηση των αλλαγών στον κώδικα από τους συναδέλφους μας μέσω των message που εξηγούν τις αλλαγές στις αποθηκευμένες εικόνες, χωρίς να χρειάζεται να κάνουμε οι ίδιοι αυτή την εξήγηση.
  • Αν επιλέξουμε να είναι οι εικόνες μας δημόσιες δίνουμε την ευκαιρία σε οποιονδήποτε να συμμετέχει και να βοηθήσει στο project μας.
  • Τα project που δημοσιοποιούνται είναι πιθανότατα πιο ασφαλή διότι η κοινότητα βοηθάει να βρεθούν λάθη.

Σε αυτόν τον οδηγό θα χρησιμοποιήσω το GitHub. Μπορείτε να δημιουργήσετε ένα git repository εκεί χρησιμοποιώντας αυτόν τον οδηγό ή σε οποιονδήποτε άλλον host θέλετε.

Το git περιέχει remotes. Ένα remote περιέχει το URL ενός online git repository. Ουσιαστικά ένα remote είναι ένας σύνδεσμος. Συνδέει το τοπικό git repository με το online git repository ώστε να μπορούμε να ανεβάσουμε τις αποθηκευμένες εικόνες μας εκεί. Το online git repository περιέχει όλες τις εικόνες που έχουμε αποφασίσει να ανεβάσουμε.

Αν εκτελέσουμε την εντολή git remote μας εμφανίζει μια λίστα από τα remotes που έχει το git repository μας.

~/cats $ git remote ~/cats $

Όπως βλέπετε δεν μας βγάζει κανένα remote γιατί δεν έχουμε δημιουργήσει κανένα. Για να δημιουργήσουμε ένα χρησιμοποιούμε την εντολή git remote add [remote_name] [url] όπου remote_name το όνομα που δίνουμε στο remote και url το URL του remote.

~/cats $ git remote add origin git@github.com:VitSalis/git-guide.git ~/cats $ git remote
origin
~/cats $ git remote -v
origin git@github.com:VitSalis/git-guide.git (fetch) origin git@github.com:VitSalis/git-guide.git (push)
~/cats $

Αν δώσουμε την παράμετρο -v στην εντολή git remote τότε μας δείχνει εκτός από το όνομα του remote και το URL του.

Τώρα έχουμε ένα online git repository και έχουμε φτιάξει ένα remote. Το μόνο που μας μένει είναι να ανεβάσουμε τις αποθηκευμένες εικόνες μας μέσω αυτού.

push

Αν μπείτε στο online git repository σας θα δείτε ότι είναι άδειο. Αυτό συμβαίνει γιατί δεν έχετε ανεβάσει ακόμα τις εικόνες σας εκεί.

Σε αυτό χρησιμεύει η εντολή git push [remote_name] master όπου remote_name το όνομα του remote που περιέχει το URL του online git repository όπου θέλουμε να ανέβουν οι εικόνες.

Η εντολή push απλά παίρνει τις αποθηκευμένες εικόνες και τις ανεβάζει στο online git repository.

~/cats $ git push origin master
Counting objects: 8, done. Delta compression using up to 4 threads. Compressing objects: 100% (8/8), done. Writing objects: 100% (8/8), 35.58 KiB | 0 bytes/s, done. Total 8 (delta 0), reused 0 (delta 0) To git@github.com:VitSalis/git-guide.git * [new branch] master -> master
~/cats $

Αν επισκεφθείτε το online git repository σας τώρα θα δείτε την τελευταία εικόνα που αποθηκεύσατε και αν κοιτάξετε τα commits θα δείτε όλες τις εικόνες που είχατε αποθηκεύσει τοπικά, με τα αντίστοιχα μηνύματα.

Αυτό ήταν! Τώρα μπορώ να μοιραστώ τις γάτες μου με όλο τον κόσμο!

Τι μάθαμε;

  • Τι είναι ένα repository.
  • Τι είναι μια εικόνα.
  • Γιατί είναι χρήσιμο να χρησιμοποιούμε το git και να ανεβάζουμε τις εικόνες μας σε online git repositories.
  • Τις εντολές init, status, add, commit, remote, push.
